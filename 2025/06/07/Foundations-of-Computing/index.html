Foundations of Computing

Date: June 7, 2025  17:11:48

Introduction

  • Part of information system
    • An information system is comprised of people, procedures, software, hardware, data, and the Internet. People are considered the most crucial component of any information system.
  • System and application software
    • Software, also known as programs, consists of instructions that direct a computer on how to process data into the desired form. There are two primary categories of software: System Software and Application Software.
    • System Software: This is the software that computers use. It enables application software to interface with the computer’s hardware and assists in managing its resources. System software is a collection of programs that includes Operating Systems and Utilities.
    • Application Software: This is software that you, the end-user, interact with. It includes general-purpose applications like word processors and spreadsheets, specialized applications like web authoring software, and mobile apps.
  • B2B, B2C, C2C
    • These are different types of electronic commerce.
    • B2C (Business-to-consumer): This involves the sale of products or services to the general public and is the most rapidly expanding form of e-commerce.
    • C2C (Consumer-to-consumer): This form of e-commerce involves transactions from one individual to another. A common example is web auctions, where buyers and sellers rarely meet in person and bids are placed electronically.
    • B2B (Business-to-business): This involves the sale of a product or service from one business to another, typically in a manufacturer-supplier relationship.

Computer Organization and Architecture

  • What is computer organization and architecture
    • Computer Architecture: Refers to the attributes of a system that a programmer can see, which directly influence the logical execution of a program. This includes the instruction set, the number of bits for data types, I/O mechanisms, and memory addressing techniques.
    • Computer Organization: Encompasses the operational units and their interconnections that implement the architectural specifications. This includes hardware details not visible to the programmer, like control signals, interfaces between the computer and peripherals, and the type of memory technology used.
  • Function
    • A computer performs four fundamental functions:
      • Data processing: Handling a wide array of data forms and processing needs.
      • Data storage: Both short-term and long-term storage.
      • Data movement: This includes input/output (I/O) for devices connected directly to the computer and data communications for remote devices.
      • Control: A control unit is responsible for managing the computer’s resources and coordinating the performance of its functional components based on instructions.
  • A top-down view of computer
    • A computer is a hierarchical system composed of interconnected subsystems. This top-down view illustrates the structure, beginning with the overall computer system, delving into the CPU and its parts, and further into the elements of the control unit.
  • Four main structural components of the computer
    • The four primary structural components are:
      • CPU (Central Processing Unit): Manages the computer’s operations and carries out data processing functions.
      • Main Memory: Stores data.
      • I/O: Facilitates data transfer between the computer and its external environment.
      • System Interconnection: A mechanism that enables communication among the CPU, main memory, and I/O.
  • CPU-Major Structural components
    • The main structural parts of the CPU include:
      • Control Unit: Manages the operations of the CPU and, by extension, the entire computer.
      • Arithmetic and Logic Unit (ALU): Carries out the computer’s data processing functions.
      • Registers: Offer storage internal to the CPU.
      • CPU Interconnection: A mechanism that allows for communication among the control unit, ALU, and registers.
  • Cache memory, register, microprocessor
    • Cache Memory: A smaller and faster type of memory situated between the processor and main memory, designed to accelerate memory access. It holds data from main memory that is likely to be needed again soon.
    • Register: Storage located inside the CPU that is used to temporarily hold data and results from ALU operations, memory addresses, or instructions being executed. In the IAS computer, registers included the Memory Buffer Register (MBR), Memory Address Register (MAR), Instruction Register (IR), and Program Counter (PC).
    • Microprocessor: The component that contains all the elements of a CPU on a single chip. It is responsible for interpreting and executing instructions.
  • IAS memory formats and partial flowchart of IAS operation
    • IAS Memory Formats: The IAS computer’s memory used 40-bit words. A number was stored as a sign bit with a 39-bit value, while an instruction word contained two separate 20-bit instructions, each with an 8-bit opcode and a 12-bit address.
    • IAS Operation Flowchart: The operation consisted of a fetch cycle and an execution cycle. The fetch cycle would retrieve the next instruction from memory using the program counter (PC) and load it into the instruction register (IR). The execution cycle would then decode the instruction and perform the necessary action.

A top-level view of computer

  • Hardware and software approaches
    • Hardware Approach: A “hardwired program” is created by physically connecting various components in a specific configuration, where data flows through a fixed sequence of arithmetic and logic functions.
    • Software Approach: This method uses a sequence of instructions or codes. The hardware includes an “instruction interpreter” that reads each instruction and generates control signals for a general-purpose set of logic functions, allowing programs to be changed without rewiring.
  • Software and major components
    • Software is a sequence of codes or instructions. The primary hardware components required for executing software are the CPU, which includes an instruction interpreter and general-purpose arithmetic and logic functions, and I/O Components for data input and output.
  • Computer component: top-level view
    • The top-level view of a computer shows the CPU, Main Memory, and I/O Modules interconnected by a System Bus. The CPU itself contains essential registers like the Program Counter (PC), Instruction Register (IR), Memory Address Register (MAR), and Memory Buffer Register (MBR).
  • Basic instruction cycle
    • The fundamental instruction cycle consists of a fetch cycle followed by an execute cycle. The processor first fetches an instruction from memory and then executes it, repeating this process until the program concludes.
  • Program execution
    • Program execution is the process of sequentially fetching and executing instructions. An instruction is retrieved from a memory location indicated by the PC and loaded into the IR. The processor interprets the instruction’s opcode and, if necessary, fetches data from memory using the address part of the instruction. The specified operation is then performed, and the PC is updated to point to the subsequent instruction.
  • Instruction cycle state diagram
    • The instruction cycle can be detailed as a series of states: instruction address calculation, instruction fetch, instruction operation decoding, operand address calculation, operand fetch, data operation, and operand store. Once an instruction is fully executed, the cycle begins again to fetch the next instruction.
  • Bus interconnection
    • A bus is a shared communication pathway that links two or more devices, such as a CPU, memory, and I/O modules. It consists of multiple lines for transferring data, addresses, and control signals. As a shared transmission medium, only one device can transmit at any given moment to prevent signal overlap and data corruption.
  • Input-output function
    • An I/O module can exchange data directly with the processor. For efficiency, I/O exchanges can also occur directly with memory through a method called direct memory access (DMA). In a DMA transfer, the processor grants the I/O module the authority to read from or write to memory, which frees the processor from managing the exchange.

Number system

  • Positional and non-positional number system
    • There are two main types of number systems: non-positional and positional.
    • Non-positional Number Systems: Use symbols where each symbol has a fixed value regardless of its position (e.g., Roman numerals). Performing arithmetic in such systems is difficult.
    • Positional Number Systems: Use a limited set of digits where the value of a digit is determined by its position within the number and the base of the system. Computers utilize positional systems like binary.
  • Convert a number’s base to another base (binary, octal, decimal and hexadecimal)
    • Another Base to Decimal: To convert a number from any base to decimal, determine the positional value of each digit, multiply the digit by its positional value, and then sum all the resulting products. For instance, $4706_8 = (4 \times 8^3) + (7 \times 8^2) + (0 \times 8^1) + (6 \times 8^0) = 2502_{10}$.
    • Decimal to Another Base: The Division-Remainder Method is used. The decimal number is repeatedly divided by the new base, and the remainders are recorded from right to left to form the new number.
    • Some Base to Another Base: This is a two-step method: first, convert the number from its original base to decimal (base 10), and second, convert the resulting decimal number to the target base.
    • Shortcut Methods:
      • Binary to Octal: Group the binary digits into sets of three starting from the right, and then convert each group to its single-digit octal equivalent.
      • Octal to Binary: Convert each octal digit to its 3-digit binary equivalent and then combine the binary groups.
      • Binary to Hexadecimal: Group the binary digits into sets of four starting from the right, and convert each group to its single-digit hexadecimal equivalent.
      • Hexadecimal to Binary: Convert each hexadecimal digit to its 4-digit binary equivalent and then combine the binary groups.

Math and Logic - Gates & Microprocessor

  • AND, OR, NOT, NAND, NOR, XOR Gate
    • NOT Gate: An inverter that produces an output that is the opposite of its single input.
    • AND Gate: Produces a 1 (high) output only when both of its inputs are 1.
    • OR Gate: Produces a 1 output if either one or both of its inputs are 1.
    • NAND Gate: (Not AND) Produces a high output unless both inputs are high.
    • NOR Gate: (Neither…NOR) Produces a high output only if neither input A nor input B is high.
    • XOR Gate: (Exclusive OR) Produces a high output if input A or input B is high, but not both.
  • Truth Table
    • A truth table is used to check the validity of algorithms, understand business rules, and design the logic for microchips. It provides an exhaustive list of all possible input combinations and their corresponding outputs for a given logic gate or circuit.

Set Theory

  • Cardinal number of a set
    • The cardinal number of a set is the number of elements contained within that set. For a set named X, this can be denoted as n(X) or |X|.
  • Types of set
    • Empty Set: A set that contains no elements, written as {}.
    • Singleton Set: A set that contains precisely one element.
    • Finite Set: A set that has a limited and countable number of elements.
    • Infinite Set: A set that has an uncountable number of elements.
  • Venn diagram, union, intersection, symmetric difference
    • Venn Diagram: A pictorial representation of sets using overlapping circles to show the relationships between them.
    • Intersection (A ∩ B): The set of elements that belong to both set A and set B.
    • Union (A ∪ B): The set containing all elements that are in set A, set B, or both.
    • Symmetric Difference (A Δ B): The set of elements that are in either of the sets, but not in their intersection.

Input and output & Secondary storage

  • Basic idea about I/O devices
    • Input Devices: These devices translate data into a form that the system unit can process. Keyboards and mice are common examples of input devices.
    • Output Devices: These devices convert data from the computer into a form that humans can understand. The most common output devices are displays (monitors) and printers.
    • Combination Input and Output Devices: These devices combine both input and output functionalities. Examples include headsets (microphone and headphones), multifunctional devices (like all-in-one printers), and virtual reality (VR) head-mounted displays and controllers.
  • Basic idea about secondary storage
    • Secondary storage is nonvolatile, meaning it holds data and programs even when the computer’s power is turned off. Its key characteristics include the media (physical material), capacity (how much it can hold), storage devices (the hardware that reads/writes data), and access speed. Common types of secondary storage are hard disks, solid-state storage (including SSDs and USB drives), and optical discs (like CDs, DVDs, and Blu-ray).

Computer language

  • Different types of language and their advantage & limitation
    • Computer languages are broadly classified into three categories: machine language, assembly language, and high-level language.
    • Machine Language:
      • Advantage: It can be executed very fast by the computer because it doesn’t require translation.
      • Limitations: It is machine-dependent, difficult to program, prone to errors, and difficult to modify.
    • Assembly Language:
      • Advantages: It is easier to understand, use, and modify than machine language, and it maintains the efficiency of machine language. It uses mnemonic codes (like ADD) instead of numeric codes.
      • Limitations: It is machine-dependent, requires knowledge of the hardware, and involves machine-level coding.
    • High-Level Language:
      • Advantages: These languages are machine-independent, easier to learn and use, result in fewer errors, have lower preparation costs, are better documented, and are easier to maintain. They use English words and familiar mathematical symbols.
      • Limitations: They have lower execution efficiency compared to low-level languages and offer less flexibility to control the computer’s CPU and memory directly.
  • Compiler and interpreter
    • Compiler: A translator program that converts an entire high-level language program (source program) into its equivalent machine language program (object program). It checks for syntax errors during this process but is not involved in the program’s execution.
    • Interpreter: A translator that takes one statement of a high-level language program at a time, translates it into machine language, and then immediately executes the resulting instructions.
  • Syntax errors
    • A syntax error is a violation of the rules of a programming language. Compilers automatically detect these errors, which can include the use of illegal characters, improper sequencing of instructions, or the use of undefined variable names. A program with syntax errors cannot be successfully translated.

Planning the computer program

  • Algorithm
    • An algorithm refers to the logic of a program and provides a step-by-step description of how to solve a given problem. To qualify as an algorithm, a sequence of instructions must be precise, unambiguous, and executable in a finite amount of time, ensuring it will eventually terminate and produce the desired result.
  • Flowchart
    • A flowchart is a pictorial or graphical representation of an algorithm. It uses standardized symbols (boxes of different shapes) to denote various types of instructions, and these boxes are connected by lines with arrows to show the sequence of execution.
  • Program and programming
    • A program is a list of instructions, written in a programming language, for a computer to follow to process data into information.
    • Programming, also known as software development, is a problem-solving procedure that involves creating this list of instructions. It typically follows a six-step process called the software development life cycle.
  • Software development life cycle (SDLC)
    • The SDLC is a six-step process for creating software. The steps are:
      1. Program specification
      2. Program design
      3. Program code
      4. Program test
      5. Program documentation
      6. Program maintenance

Networking

  • Basic idea about network and how does it works
    • A network is used to exchange data for purposes such as sharing, communication, organization, and entertainment. A network can be a client-server model, where client machines request services from a central server, or a peer-to-peer network, where computer nodes can act as both clients and servers without a central server.
  • Request and response
    • This is a model of communication used in client-server networks. A client machine sends a request for information or a service to a server. The server processes this request and sends a response back to the client.
  • MAC, LAN, MAN, WAN, PAN, VPN, TCP/IP, UDP, Multiplexing, IPv4, IPv6, subnetting
    • MAC (Media Access Control) address: A unique identifier assigned to a network adapter by its manufacturer, written as six octets in hexadecimal.
    • LAN (Local Area Network): A group of PCs confined to a small geographic area, like a single building or floor.
    • MAN (Metropolitan Area Network): A telecommunication network that connects several LANs in close proximity.
    • WAN (Wide Area Network): A network that extends over large geographic areas, such as countries or continents.
    • PAN (Personal Area Network): A network for integrating personal devices like smartphones and laptops for data exchange.
    • VPN (Virtual Private Network): A virtual communication network that uses the infrastructure of a physical network, most commonly the internet, to logically connect computer systems.
    • TCP/IP (Transmission Control Protocol/Internet Protocol): A communications protocol suite where IP works at OSI Layer 3 to get packets to their destination, and TCP works at Layer 4 to provide reliable, connection-oriented data transmission.
    • UDP (User Datagram Protocol): An OSI Layer 4 protocol that is a “cut-down” version of TCP. It is connectionless and does not guarantee delivery, making it faster but less reliable, suitable for real-time applications.
    • Multiplexing: A technique that combines multiple signals into a single transmission channel to enable efficient sharing of a communication medium.
    • IPv4: Internet Protocol version 4, the most frequently used communications protocol, which uses a 32-bit address system.
    • IPv6: The new generation of IP addressing, which uses a 128-bit system to overcome the address space limitations of IPv4.
    • Subnetting: The subdivision of a logical IP network into multiple smaller networks by modifying the default subnet mask.
  • Nodes, repeaters, hubs, bridges, switch, port, firewalls
    • Nodes: Computers or other intelligent devices with MAC addresses that can send and receive messages.
    • Repeaters: Devices that boost weak digital signals to extend their range.
    • Hubs: Central connecting devices that connect hosts and broadcast any received data to all other connected hosts.
    • Bridges: Two-port devices used to create subnets that can process and filter data.
    • Switch: A multiport device similar to a hub, but it forwards messages to a specific port based on the destination MAC address.
    • Port Numbers: 16-bit unsigned integers (0-65535) that operate at the transport layer to identify different applications or services running on a computer.
    • Firewalls: Devices that use software to control services and convert between protocols, often incorporated into routers to block unwanted network connections. They act as a security buffer between a private network and external networks.
  • Network topologies
    • A network’s topology refers to the physical or logical arrangement of its nodes. Common topologies include:
      • Star: Hosts are connected to a central connecting device like a hub or switch.
      • Bus: All nodes are connected in a line to a single cable.
      • Ring: Messages are passed from computer to computer in a logical ring.
      • Mesh: Each node is connected to other nodes, allowing it to send, receive, and relay data.
      • Tree & Hybrid: Other topologies that combine aspects of the basic types.
  • How find links for nodes
    • To find the number of links required for a full mesh topology with ‘n’ nodes, you can use the formula: n(n - 1) / 2. For example, a network with 6 nodes would require 6(6-1)/2 = 15 links.
  • OSI reference model
    • The Open Systems Interconnection (OSI) model is a 7-layer framework used to define how data communication occurs on computer networks. Each layer provides services to the layers above and below it. The layers are: 7. Application, 6. Presentation, 5. Session, 4. Transport, 3. Network, 2. Data Link, and 1. Physical.
  • Protocol by layer
    • Different protocols operate at each layer of the OSI model. For example:
      • Layer 7 (Application): SMTP, FTP, Telnet.
      • Layer 4 (Transport): TCP, UDP.
      • Layer 3 (Network): IP, IPv4, IPv6, Routers.
      • Layer 2 (Data Link): Switches, Bridges, MAC addresses, Frames.
      • Layer 1 (Physical): Hubs, Cables, Bits.
  • Packet switching
    • Packet switching is a communication method where data is broken down into smaller units called packets. Each packet contains source and destination addresses and is transmitted over the network, potentially taking different paths to reach the destination where they are reassembled.
  • Class of IP (A-E) and range
    • The IPv4 classification system is known as the classful network architecture, which divides IP addresses into five classes. Classes A, B, and C are commonly used for network hosts.
      • Class A: Range 0-127 for the first octet. Default subnet mask 255.0.0.0.
      • Class B: Range 128-191 for the first octet. Default subnet mask 255.255.0.0.
      • Class C: Range 192-223 for the first octet. Default subnet mask 255.255.255.0.
      • Class D: Range 224-239, used for multicasting.
      • Class E: Range 240-255, was reserved for future use.

Privacy, security and ethics

  • Authentication and authorization
    • Authentication is the process of ensuring the integrity and identity of a user. Methods include passwords and biometric scanning (fingerprints, iris scans).
    • Authorization is not explicitly defined in the documents but is related to restricting access, which involves limiting access to authorized persons.
  • Define privacy and discuss the impact of large databases, private networks, the Internet, and the Web.
    • Privacy concerns the collection and use of data about individuals. Key concerns include accuracy, property (who owns the data), and access.
    • Impact of Large Databases: Large organizations compile information about individuals, creating “Big Data”. Information resellers collect and sell this personal data, creating electronic profiles. This raises issues like the collection of public but personally identifying information, the spread of information without consent (which can lead to identity theft), and the spread of inaccurate data.
    • Impact of Private Networks: Employers can use employee-monitoring software to monitor emails legally, which is a significant privacy concern.
    • Impact of the Internet and the Web: People often have an “illusion of anonymity” online. When browsing, information is stored in history files, temporary internet files (browser cache), and cookies. Tracking cookies and spyware can monitor internet activities, and computer monitoring software like keystroke loggers are the most invasive threats.
  • identity and the major privacy laws
    • Online Identity is the information that people voluntarily post about themselves online. This information can be archived and made available indefinitely through search features.
    • Major Privacy Laws:
      • Gramm-Leach-Bliley Act: Protects personal financial information.
      • Health Insurance Portability and Accountability Act (HIPAA): Protects medical records.
      • Family Educational Rights and Privacy Act (FERPA): Restricts the disclosure of educational records.
  • malicious software, and malicious hardware including identity theft, Internet scams, data manipulations, ransomware, DoS attacks, viruses, worms, Trojan horses, zombies
    • Identity Theft: The illegal assumption of a person’s identity for economic gain.
    • Internet Scams: Scams conducted over the Internet, such as phishing, which uses fake but official-looking communications to trick people into giving up sensitive data.
    • Data Manipulation: Unauthorized access to a computer network to copy files to or from a server.
    • Ransomware: Malicious software that encrypts a computer’s data and demands a ransom for the password to decrypt it.
    • DoS (Denial of Service) Attacks: Attempts to slow down or stop a computer system by flooding it with requests for information and data.
    • Malicious Software (Malware):
      • Viruses: Migrate through networks and attach themselves to different programs.
      • Worms: Fill a computer with self-replicating information to overwhelm it.
      • Trojan Horse: Programs disguised as something else that are actually malicious.
    • Malicious Hardware:
      • Zombies: Computers that have been infected by a virus, worm, or Trojan horse and can be controlled remotely. A collection of zombies is a botnet.
      • Rogue Wi-Fi Hotspots: Fake Wi-Fi networks that imitate legitimate ones to capture any data that passes through them.
